# LucidShell UI 设计

## 1. 设计目标

LucidShell 的界面应该服务于长时间、高频率的 SSH 工作。

整体风格：

- 美观、简洁、专业。
- 信息密度适中，方便扫描。
- 避免花哨装饰，优先保证稳定、清楚、易用。
- 默认支持深色主题，后续支持浅色主题。
- 所有关键操作都应该有明确的 hover、active、disabled、loading 状态。

## 2. 整体布局

主界面由 5 个区域组成：

- 顶部自绘窗口栏。
- 左侧连接栏。
- 主工作区。
- Terminal/SFTP 上下分栏。
- 底部状态栏。

默认布局：

```text
+--------------------------------------------------------------+
| LucidShell                                      Theme Settings|
+----------------+---------------------------------------------+
| 连接  ↓ ↑ + ↻ <<| Terminal Tabs: shell-1 deploy logs +         |
| Search...  + ≡ +---------------------------------------------+
| Favorites      | Terminal                                    |
| prod-01        |                                             |
| staging        |                                             |
|                +---------------------------------------------+
| Recent         | SFTP: /var/www/app                           |
| dev-box        | File table                                   |
| local-vm       |                                             |
+----------------+---------------------------------------------+
| prod-01 · root@10.0.0.12 · 32ms · CPU 18% · MEM 42% · SFTP ok |
+--------------------------------------------------------------+
```

尺寸建议：

- 顶部窗口栏高度：`40px` 到 `44px`。
- 左侧连接栏展开宽度：`240px` 到 `280px`。
- 左侧连接栏折叠宽度：`48px` 到 `56px`。
- 底部状态栏高度：`24px` 到 `28px`。
- Terminal 最小高度：`240px`。
- SFTP 最小高度：`180px`。

## 3. 顶部自绘窗口栏

顶部窗口栏只保留一行，保持克制。

左侧内容：

- Logo。
- 应用名：`LucidShell`。

右侧内容：

- 主题切换按钮。
- 设置按钮。
- 最小化按钮。
- 最大化/还原按钮。
- 关闭按钮。

示意：

```text
LucidShell                                      ◐  ⚙  −  □  ×
```

交互要求：

- 非按钮区域可拖拽窗口。
- 按钮区域不可拖拽窗口。
- 窗口控制按钮需要符合当前平台习惯。
- 关闭按钮 hover 时应有明显但克制的危险状态。
- 顶部栏不放搜索、服务器摘要、Tab 操作或复杂工作流入口。

## 4. 左侧连接栏

左侧连接栏负责连接资产管理和服务器切换。

展开状态结构：

```text
连接                            ↓  ↑  +  ↻  <<
+-------------------------------------------+
| 搜索...                              +  ≡ |
+-------------------------------------------+

Favorites
  prod-01
  staging

Groups
  personal
  work

Recent
  dev-box
  local-vm
```

标题行：

- 左侧显示 `连接`。
- 右侧显示紧凑图标按钮组。
- 所有图标按钮必须有 tooltip。
- 按钮使用统一尺寸，建议 `28px` 到 `32px`。

标题行按钮：

- 导入连接。
- 导出连接。
- 新增分组。
- 刷新连接列表。
- 折叠左侧连接栏。

推荐图标：

- 导入连接：`Download` 或 `ArrowDownToLine`。
- 导出连接：`Upload` 或 `ArrowUpFromLine`。
- 新增分组：`FolderPlus`。
- 刷新连接列表：`RefreshCw`。
- 折叠左侧连接栏：`PanelLeftClose`。

搜索行：

- 左侧是连接搜索输入框。
- 右侧可以放新增连接和筛选/排序按钮。
- 搜索输入框 placeholder：`搜索...`。
- 搜索结果应覆盖收藏、分组和最近连接。

连接项内容：

- 在线状态点。
- 连接名称。
- 用户名和主机，例如 `root@10.0.0.12`。
- hover 时显示更多操作。
- 当前选中连接要有清晰背景。

连接项右键菜单：

```text
连接
---
打开连接
在新窗口打开
编辑连接
复制连接
---
新增到收藏 / 从收藏移除
移动到分组
导出此连接
---
删除连接
```

MVP 阶段建议包含：

- 打开连接。
- 编辑连接。
- 复制连接。
- 新增到收藏 / 从收藏移除。
- 移动到分组。
- 删除连接。

后续可以补充：

- 在新窗口打开。
- 导出此连接。
- 连接测试。
- 查看连接历史。

交互要求：

- 危险操作，例如删除连接，需要使用危险色，并弹出确认。
- 删除连接不应删除已导出的配置文件。
- 编辑连接时，如果该连接已经在线，需要提示修改会在下次连接时生效。
- 菜单项根据当前状态启用或禁用，例如已连接时可以显示断开连接。

折叠状态：

```text
LS
⌕
+
★
◷
```

折叠状态要求：

- 宽度保持在 `48px` 到 `56px`。
- 显示应用标识或连接入口图标。
- 显示搜索、添加连接、收藏、最近等核心入口。
- 当前连接状态可以用小圆点表达。
- hover 图标时显示 tooltip。
- 折叠状态需要持久化到本地。

## 5. Terminal Tabs

Terminal Tabs 位于主工作区顶部。

结构：

```text
shell-1   deploy   logs   +
```

每个 Tab 表示一个完整工作上下文。

Tab 需要保存：

- Terminal 会话。
- SFTP 当前路径。
- SFTP 文件选择状态。
- SFTP 展开/折叠状态。
- Terminal/SFTP 分栏比例。
- 传输任务摘要。

交互要求：

- 点击切换 Tab。
- `+` 新建 Terminal Tab。
- Tab 支持关闭。
- Tab 支持重命名。
- 当前 Tab 有明确选中态。
- 有传输任务或错误时，可以在 Tab 上显示小状态点。

Terminal Tab 右键菜单：

```text
shell-1
---
重命名
复制 Tab
关闭 Tab
关闭其他 Tab
关闭右侧 Tab
---
重新连接
断开连接
---
显示/隐藏 SFTP
重置分栏比例
打开当前路径的 SFTP
```

MVP 阶段建议包含：

- 重命名。
- 关闭 Tab。
- 关闭其他 Tab。
- 显示/隐藏 SFTP。
- 重置分栏比例。
- 重新连接。
- 断开连接。

后续可以补充：

- 复制 Tab。
- 关闭右侧 Tab。
- 打开当前路径的 SFTP。
- 拆分 Terminal。
- 将 Tab 移动到新窗口。

交互要求：

- 关闭正在运行的 Tab 时，如果存在活跃进程或未完成传输，需要二次确认。
- 断开连接后，Tab 保留终端输出历史，并显示可重新连接状态。
- 显示/隐藏 SFTP 操作只影响当前 Terminal Tab。
- 重置分栏比例恢复到默认 Terminal `62%`、SFTP `38%`。

## 6. Terminal 区域

Terminal 区域使用 xterm.js 渲染。

要求：

- 占据主工作区上半部分。
- 默认聚焦当前激活 Tab 的 terminal。
- 切换 Tab 后恢复对应 terminal 状态。
- 尺寸变化时自动调用 xterm fit。
- 支持复制、粘贴、选择文本。
- 支持深色主题下足够高的对比度。

空状态：

- 未连接服务器时，主区域显示连接引导。
- 连接中时，显示轻量 loading。
- 连接失败时，显示错误原因和重试按钮。

## 7. Terminal/SFTP 上下分栏

Terminal 和 SFTP 使用上下分栏。

默认比例：

- Terminal：`62%`。
- SFTP：`38%`。

分隔条：

- 高度建议 `4px` 到 `6px`。
- 支持拖拽调整高度。
- hover 时显示可拖拽状态。
- 双击分隔条可以折叠或恢复 SFTP。

约束：

- Terminal 最小高度：`240px`。
- SFTP 最小高度：`180px`。
- 每个 Terminal Tab 独立保存分栏比例。

## 8. SFTP 面板

SFTP 面板属于当前 Terminal Tab。

展开状态结构：

```text
SFTP  /var/www/app                         ↑  ↓  +  ↻  ...
+---------------------------------------------------------+
| Name                 Size      Modified           Mode   |
| public               -         2026-06-13 10:12   drwx   |
| app.log              12.4 MB   2026-06-13 10:11   -rw-   |
+---------------------------------------------------------+
```

顶部栏内容：

- `SFTP` 标识。
- 当前路径或面包屑。
- 上传按钮。
- 下载按钮。
- 新建文件夹按钮。
- 刷新按钮。
- 更多操作按钮。
- 折叠按钮。

文件列表列：

- 文件名。
- 大小。
- 修改时间。
- 权限。

交互要求：

- 双击文件夹进入目录。
- 双击文件可以下载或打开远程编辑，MVP 可先下载。
- 支持多选。
- 支持右键菜单。
- 支持拖拽上传，后续版本可以补充。
- 文件操作失败时显示行内错误或 toast。

折叠状态：

```text
SFTP  /var/www/app  ·  2 selected  ·  transfer 42%          ↑
```

折叠状态要求：

- 保留一条 `32px` 到 `40px` 的状态条。
- 显示当前远端路径。
- 显示选中文件数量。
- 显示传输状态摘要。
- 右侧提供展开按钮。
- 每个 Terminal Tab 独立保存折叠状态。

## 9. 底部状态栏

底部状态栏展示当前服务器上下文。

示例：

```text
prod-01 · root@10.0.0.12 · connected · 32ms · CPU 18% · MEM 42% · transfers 2
```

内容：

- 当前服务器名称。
- 用户名和主机。
- 连接状态。
- 延迟。
- CPU 使用率。
- 内存使用率。
- SFTP 状态。
- 传输任务摘要。
- 必要时显示 shell、编码或当前目录。

要求：

- 高度低，占用空间小。
- 文本要可快速扫描。
- 重要异常状态可以使用强调色。
- 不放主要操作按钮。

## 10. 颜色与主题

默认主题建议为深色专业主题。

深色主题建议：

- 应用背景：接近黑色但不要纯黑。
- 面板背景：深灰。
- 分隔线：低对比灰色。
- 主文本：高对比浅色。
- 次文本：中对比灰色。
- 强调色：用于选中态、焦点态和关键状态。
- 危险色：只用于删除、关闭、连接失败等场景。

浅色主题后续补充。

主题原则：

- 不使用大面积渐变。
- 不使用装饰性光斑或复杂背景。
- 不使用单一色系铺满界面。
- 长时间阅读和操作时不刺眼。

## 11. 字体与图标

字体：

- UI 字体使用系统字体或 Inter。
- Terminal 字体使用 JetBrains Mono、Cascadia Code 或用户配置字体。

图标：

- 优先使用 lucide 图标。
- 工具按钮优先使用图标，不使用文字按钮。
- 不熟悉的图标必须有 tooltip。
- 图标按钮尺寸和间距保持一致。

文字：

- 中文界面优先使用简洁短标签。
- 按钮文字避免过长。
- 状态栏文字使用短语，不写完整说明句。

## 12. 状态设计

需要覆盖以下状态：

- 未连接。
- 连接中。
- 已连接。
- 连接失败。
- 断线重连中。
- SFTP 加载中。
- SFTP 空目录。
- SFTP 操作失败。
- 上传中。
- 下载中。
- 传输完成。
- 传输失败。

错误提示原则：

- 错误信息要说明发生了什么。
- 能重试的地方提供重试入口。
- 不把底层错误原样堆给用户，但保留可展开详情。

## 13. 响应式与最小尺寸

桌面端优先。

推荐最小窗口尺寸：

- 宽度：`960px`。
- 高度：`640px`。

小窗口策略：

- 左侧连接栏自动建议折叠。
- SFTP 可以保持折叠状态。
- Terminal 保持优先空间。
- 文件列表列可以减少，只保留文件名、大小、修改时间。

布局要求：

- 文本不能互相遮挡。
- 按钮尺寸不能因内容变化而跳动。
- Tab、状态栏、文件列表在窄屏下要有截断和 tooltip。
- Terminal resize 后必须重新 fit。
