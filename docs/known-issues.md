# LucidShell 已知问题

本文档记录当前 Beta 阶段的已知限制、风险和后续计划。这里的问题不一定都会阻塞使用，但需要在分发和测试时提前说明。

## 分发与安装

- **安装包尚未代码签名。**  
  Windows 可能提示“未知发布者”或安全警告。

- **自动更新尚未接入。**  
  新版本需要重新下载安装包。

- **尚未建立 CI 自动发布流程。**  
  当前安装包仍依赖本地 `npm run tauri build` 生成。

- **安装包只在当前 Windows 开发环境验证过。**  
  还需要在干净 Windows 用户环境、不同 Windows 版本上继续测试。

## Terminal

- **连接健康检查是轻量状态同步。**  
  当前主要依赖 russh keepalive、session closed 状态和实际读写失败来感知断线。极端网络半断场景下，状态可能不会立刻变更。

- **非 UTF-8 服务器可能乱码。**  
  当前终端输出按 UTF-8 lossy 方式处理，GBK、Big5 或其他 legacy locale 还没有专门支持。

- **特殊终端控制序列尚未系统性压测。**  
  常规 shell、vim/top 等场景可继续验证，但复杂控制序列、鼠标模式、全屏 TUI 程序仍需要更多测试。

- **大文本粘贴已做保护，但仍需真实场景压测。**  
  当前有大文本确认、写入队列、分片发送，但远端 shell 处理能力仍取决于服务器和当前程序。

## SFTP

- **超大目录和海量小文件传输仍需压测。**  
  文件夹上传/下载已经支持递归，但几千到几万个文件的性能、取消和错误恢复还需要更多验证。

- **传输失败后的恢复策略仍较基础。**  
  当前会记录错误并停止对应任务，尚未支持断点续传或失败项重试队列。

- **路径冲突策略还不完整。**  
  上传到已有同名文件/目录时，当前主要依赖远端 SFTP 行为，尚未提供覆盖、跳过、重命名等交互策略。

- **权限错误提示还可以更细。**  
  当前会显示后端返回的错误文本，但没有针对权限不足、磁盘满、路径不存在等情况做专门文案。

## 本地编辑

- **本地编辑缓存清理策略还不完整。**  
  目前文件会放在 app cache 下，后续需要增加过期清理或手动清理入口。

- **多编辑器保存行为仍需测试。**  
  不同编辑器保存文件时可能触发不同的文件事件，例如临时文件替换、原子保存、延迟写入。

- **编辑冲突暂未处理。**  
  如果远端文件在本地编辑期间被其他人修改，当前不会检测冲突。

## 凭据与安全

- **系统凭据需要更多 Windows 环境验证。**  
  当前使用 keyring 保存系统凭据，但不同系统策略、凭据服务状态可能影响保存和读取。

- **明文密码保存是有意保留的 Beta 能力。**  
  用户选择明文保存时，导出连接会包含密码。正式分发前需要更明显的风险提示和文档说明。

- **私钥登录还不是完整闭环。**  
  当前架构已有 private key 字段和后端能力雏形，但产品侧主要验证路径仍是密码登录。

## 诊断与日志

- **诊断日志可能包含路径、主机、用户名等上下文。**  
  当前不会主动写入密码或私钥内容，但导出日志前仍建议用户检查敏感路径或服务器信息。

- **日志轮转尚未实现。**  
  前端只保留最近 300 条展示，后端 `diagnostics.log` 目前会持续追加。后续需要增加大小限制或轮转。

- **日志导出不包含系统环境完整快照。**  
  当前包含版本、构建信息、错误上下文。后续可以补充 OS/WebView/Rust 后端版本等信息。

## UI 与体验

- **Vite 构建存在 chunk 超过 500KB 的提示。**  
  不影响运行，但后续可以通过动态导入或 manualChunks 优化。

- **小屏幕体验仍需继续打磨。**  
  左侧连接栏、Terminal/SFTP 分栏、上传下载队列在低分辨率下需要更多实际验证。

- **部分错误仍显示原始后端文本。**  
  Beta 阶段有助于排障，但正式版应逐步替换为更友好的错误文案。

## 后续优先级建议

1. 干净 Windows 环境安装测试
2. 系统凭据跨机器验证
3. SFTP 大文件与海量小文件压测
4. 日志轮转和缓存清理
5. 路径冲突处理
6. 自动更新
7. 代码签名
8. CI 自动构建和 GitHub Release 发布
